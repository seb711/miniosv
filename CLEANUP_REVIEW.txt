REPO CLEANUP — aggressive dead-file removal for a non-Linux, lightweight kernel
================================================================================

Goal: keep ONLY what is essential to build + boot the slim kernel. This is a
NON-Linux-compatible, statically-linked, single-in-kernel-app OSv. There is no
ELF loader, no syscall ABI, no rich VFS (only a read-only rofs data disk), no
networking, no FreeBSD/ZFS, no GNU toolchain (clang + llvm-libc + libc++ only).

Method (transitive rule): a file is kept ONLY if its dependency chain reaches a
REAL consumer — the kernel+app build (`scripts/build fs=rofs image=os-features`),
`scripts/run.py`, the aarch64 cross build, or legal (LICENSE). A chain that only
loops back on libc API surface that nothing calls is dead and removed whole.

Every removal below was validated: x64 from-scratch `scripts/build fs=rofs
image=os-features`, `run.py -c2 -m 1G` => 26/26 passed, exit 0, and 0 real
syscall instructions (`0f 05`) in loader.elf.

libc/misc/uname.c  — REWRITTEN (de-Linux'd), NOT deleted
    Almost dead — uname() has no callers — BUT it also defines the global
    `struct utsname utsname`, which libc/unistd/sethostname.c references (and is
    the host-name storage). So it cannot be deleted outright (link error:
    "undefined reference to utsname"). Rewritten to drop the <linux/version.h>
    coupling and the "Linux 3.7.0" lie; now reports sysname "OSv". Kept its
    `libc += misc/uname.o` Makefile line.

    Unreferenced. The rofs image (usr_rofs.manifest.skel) only pulls
    static/etc/hosts and the build-generated `fstab` (copied from fstab_rofs).
    KEPT: static/etc/hosts, static/etc/fstab_rofs (both build-essential).

dummy-shlib.cc  — ESSENTIAL (re-verified empirically THIS session)
    Your note: "we have a fixed binary and no elf reloc's anymore, just remove
    all of this." This is incorrect, and I tested it to be sure: I removed it
    from stage1_targets, rebuilt, and booted.
      RESULT: loader.elf links with NO .dynamic / .dynsym section, and the
      kernel ABORTS at boot ("OSv v0.57... / Aborted / Halting", run.py exit
      124 / timeout). Restoring it -> .dynamic present, 26/26, exit 0.
    Why: even though the kernel is non-PIE, OSv self-relocates at boot via
    .rela.dyn / DT_RELA, which requires the .dynamic section. Linking against a
    do-nothing .so is the ld workaround that forces ld to keep .dynamic/.dynsym.
    (dl_iterate_phdr is also live: libunwind walks program headers to find
    .eh_frame for C++ exception unwinding — Phase 8.7.) DO NOT REMOVE.
